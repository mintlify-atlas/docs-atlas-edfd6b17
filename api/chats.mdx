## Overview

Chat endpoints enable group coordination for venue visits. When 2+ users confirm interest in an action item, a group chat is automatically created for planning logistics.

## Create Chat

Creates a new group chat for coordinating a venue visit.

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8000/chats" \
  -H "Content-Type: application/json" \
  -d '{
    "action_item_id": "action_item_123",
    "venue_id": "venue_1",
    "participant_ids": ["user_1", "user_2", "user_3"]
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8000/chats",
    json={
        "action_item_id": "action_item_123",
        "venue_id": "venue_1",
        "participant_ids": ["user_1", "user_2", "user_3"]
    }
)
print(response.json())
```
</CodeGroup>

### Request Body

<ParamField body="action_item_id" type="string" required>
  The action item ID that triggered this chat
</ParamField>

<ParamField body="venue_id" type="string" required>
  The venue this chat is coordinating around
</ParamField>

<ParamField body="participant_ids" type="array" required>
  List of user IDs who are participants in this chat
</ParamField>

### Response

<ResponseField name="id" type="string">
  Unique identifier for the created chat
</ResponseField>

<ResponseField name="action_item_id" type="string">
  Associated action item ID
</ResponseField>

<ResponseField name="venue_id" type="string">
  Associated venue ID
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of chat creation
</ResponseField>

### Example Response

```json
{
  "id": "chat_abc123",
  "action_item_id": "action_item_123",
  "venue_id": "venue_1",
  "created_at": "2024-03-15T10:30:00Z"
}
```

## Get Chat Messages

Retrieves all messages in a specific chat, ordered chronologically.

<CodeGroup>
```bash cURL
curl "http://localhost:8000/chats/chat_abc123/messages"
```

```python Python
import requests

response = requests.get("http://localhost:8000/chats/chat_abc123/messages")
messages = response.json()
```
</CodeGroup>

### Path Parameters

<ParamField path="chat_id" type="string" required>
  The unique identifier of the chat
</ParamField>

### Response

Returns an array of message objects:

<ResponseField name="messages" type="array">
  Array of message objects
  
  <Expandable title="Message object">
    <ResponseField name="id" type="string">
      Unique message identifier
    </ResponseField>
    
    <ResponseField name="chat_id" type="string">
      Parent chat ID
    </ResponseField>
    
    <ResponseField name="sender_id" type="string">
      User ID of the message sender
    </ResponseField>
    
    <ResponseField name="content" type="string">
      Message text content
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when message was sent
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Response

```json
[
  {
    "id": "msg_1",
    "chat_id": "chat_abc123",
    "sender_id": "user_1",
    "content": "What time should we meet?",
    "created_at": "2024-03-15T10:35:00Z"
  },
  {
    "id": "msg_2",
    "chat_id": "chat_abc123",
    "sender_id": "user_2",
    "content": "How about 7pm?",
    "created_at": "2024-03-15T10:36:30Z"
  }
]
```

## Send Message

Sends a new message to a chat.

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8000/chats/chat_abc123/messages" \
  -H "Content-Type: application/json" \
  -d '{
    "sender_id": "user_1",
    "content": "Sounds good! See you then."
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8000/chats/chat_abc123/messages",
    json={
        "sender_id": "user_1",
        "content": "Sounds good! See you then."
    }
)
print(response.json())
```
</CodeGroup>

### Path Parameters

<ParamField path="chat_id" type="string" required>
  The unique identifier of the chat
</ParamField>

### Request Body

<ParamField body="sender_id" type="string" required>
  User ID sending the message (must be a chat participant)
</ParamField>

<ParamField body="content" type="string" required>
  Message text content (1-1000 characters)
</ParamField>

### Response

<ResponseField name="id" type="string">
  Unique message identifier
</ResponseField>

<ResponseField name="chat_id" type="string">
  Parent chat ID
</ResponseField>

<ResponseField name="sender_id" type="string">
  User ID of sender
</ResponseField>

<ResponseField name="content" type="string">
  Message content
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp
</ResponseField>

### Example Response

```json
{
  "id": "msg_3",
  "chat_id": "chat_abc123",
  "sender_id": "user_1",
  "content": "Sounds good! See you then.",
  "created_at": "2024-03-15T10:37:15Z"
}
```

## Notes

<Note>
  **Auto-creation**: Chats are automatically created when 2+ users confirm interest in an action item via the "Go Ahead" flow. Manual chat creation is also supported.
</Note>

<Info>
  **Polling**: The iOS app polls for new messages every 5 seconds while a chat view is active to provide real-time updates.
</Info>

<Warning>
  **Participants only**: Only users listed in `participant_ids` can send messages to a chat. Attempting to send from a non-participant returns a 403 error.
</Warning>
