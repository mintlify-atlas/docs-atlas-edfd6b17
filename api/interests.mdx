## Express Interest in Venue

<CodeGroup>
```bash cURL
curl -X POST "http://localhost:8000/interests" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_1",
    "venue_id": "venue_1"
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8000/interests",
    json={
        "user_id": "user_1",
        "venue_id": "venue_1"
    }
)
print(response.json())
```
</CodeGroup>

### Request Body

<ParamField body="user_id" type="string" required>
  User ID expressing interest. Must be non-empty, max 100 characters, alphanumeric and underscore only.
</ParamField>

<ParamField body="venue_id" type="string" required>
  Venue ID to express interest in. Must be non-empty, max 100 characters, alphanumeric and underscore only.
</ParamField>

### Behavior

This endpoint **toggles** interest:

- **If user is NOT interested**: Adds interest, creates activity for friends' social feeds, and may create an action item if 5+ users are interested
- **If user IS already interested**: Removes interest and deletes associated activity

## Activity Creation

When a user expresses interest in a venue, the system:

1. **Creates an Interest Record**: Links the user to the venue
2. **Creates an Activity**: Friends see this in their social feed
3. **Evaluates Action Item Creation**: If 5+ users are interested, creates an action item

Activities are committed separately from action items to ensure they always persist, maintaining consistency across Profile, Discover, Recommended, and Social tabs.

## Action Item Threshold

Action items are automatically created when:
- **5 or more users** express interest in the same venue
- No active action item already exists for that venue

The action item:
- Expires after **90 days** from creation
- Can be completed, dismissed, or initiated for group coordination
- Notifies all interested users about the opportunity to coordinate
