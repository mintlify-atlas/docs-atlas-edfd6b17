## Get Personalized Recommendations

<CodeGroup>
```bash cURL
curl "http://localhost:8000/recommendations?user_id=user_1"
```

```python Python
import requests

response = requests.get(
    "http://localhost:8000/recommendations",
    params={"user_id": "user_1"}
)
print(response.json())
```
</CodeGroup>

### Query Parameters

<ParamField query="user_id" type="string" required>
  User ID to generate personalized recommendations for
</ParamField>

### Recommendation Algorithm

Recommendations are scored based on **other users' interests only** (excluding the current user). This ensures venue positions don't change when the user toggles their own interest.

**Scoring Factors (out of 10 points):**

- **Popularity (30% = 3.0 points)** - Based on other users interested (excluding current user)
- **Category Match (25% = 2.5 points)** - Match with user's interest categories
- **Friend Interest (25% = 2.5 points)** - Number of friends interested
- **Proximity (20% = 2.0 points)** - Distance from user location

Venues are sorted by total score in descending order. Venues the user is already interested in are included with an `already_interested` flag.

## Score Stability

**Critical Behavior**: Recommendation scores are calculated based on **OTHER users' interests only**, excluding the current user. This ensures:

- Venue positions remain stable when the user toggles their interest
- The `already_interested` flag changes, but scores don't
- Users can easily find venues they've previously marked as interesting

## Proximity Scoring

Proximity is calculated using the Haversine distance formula:
- **≤ 2 km**: Maximum proximity score + added to reason string
- **2-10 km**: Gradually decreasing score
- **> 10 km**: Minimal proximity score
