## Overview

This quickstart will guide you through setting up Luna's backend and iOS app to explore the demo with 4 pre-configured users and 20+ venues across NYC.

<Note>
  **Time estimate**: 8-10 minutes
  
  **Prerequisites**: macOS Ventura+, Xcode 15.0+, Python 3.10+
</Note>

## Setup workflow

<Steps>
  <Step title="Start the backend server">
    The backend runs on FastAPI and uses SQLite for persistence. An automated setup script handles all configuration.

    ```bash
    # Navigate to backend directory
    cd Luna-Backend

    # Run automated setup (creates venv, installs deps, seeds database)
    chmod +x setup.sh && ./setup.sh

    # Activate virtual environment
    source venv/bin/activate

    # Start development server with auto-reload
    uvicorn main:app --reload --port 8000
    ```

    You should see:
    ```bash
    INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
    INFO:     Started reloader process
    INFO:     Started server process
    INFO:     Waiting for application startup.
    INFO:     Application startup complete.
    ```

    <Note>
      The setup script automatically creates `luna.db` with 20+ venues, 4 demo users, bidirectional friendships, and sample activity data.
    </Note>

    **Verify the API is running**:
    
    Visit [http://localhost:8000/docs](http://localhost:8000/docs) to see interactive Swagger documentation.
  </Step>

  <Step title="Configure the iOS app">
    Open the Xcode project and update the API base URL:

    ```bash
    # Open Xcode project
    open name.xcodeproj
    ```

    In Xcode, navigate to `name/Services/APIService.swift` and update the base URL:

    ```swift name/Services/APIService.swift
    // MARK: - Configuration
    
    // For iOS Simulator
    private let baseURL = "http://localhost:8000"
    
    // For physical iOS device (use your Mac's local IP)
    // private let baseURL = "http://192.168.1.XXX:8000"
    ```

    <Warning>
      **Physical devices**: You must use your Mac's local IP address instead of `localhost`. Find it in **System Settings → Wi-Fi → Details → IP Address**
    </Warning>
  </Step>

  <Step title="Build and run the iOS app">
    In Xcode:

    1. Select target device: **iPhone 15 Pro** (simulator) or your physical iOS 17+ device
    2. Press **Cmd+R** to build and run
    3. Wait for the build to complete (~30-60 seconds on first build)

    The app will launch and display the user switcher screen.
  </Step>

  <Step title="Explore with demo users">
    Luna includes 4 demo users with different preference profiles:

    **Alex Chen** - Coffee connoisseur  
    Interests: Coffee shops, cultural venues  
    Location: Times Square area

    **Jordan Kim** - Food blogger  
    Interests: Restaurants, bars, food culture  
    Location: East Village

    **Morgan Taylor** - Night owl  
    Interests: Bars, entertainment, nightlife  
    Location: Chelsea

    **Sam Rivera** - Outdoor enthusiast  
    Interests: Parks, outdoor venues, cultural sites  
    Location: Upper West Side

    <Tip>
      Select different users to see how recommendations and social feeds change based on their preferences and friend connections!
    </Tip>
  </Step>
</Steps>

### For You tab

View personalized recommendations with:

- **Match meter visualization** - Animated progress rings showing 0-10 scores
- **"Why This?" callouts** - Transparent explanations for each recommendation
- **Score breakdowns** - Tap scores to see popularity, category, friend, and proximity weights
- **Pull-to-refresh** - Re-rank recommendations with new data

```swift
// Match meter color tiers
Excellent (8-10): Green gradient
Great (6-8):      Blue gradient  
Good (4-6):       Orange gradient
Fair (0-4):       Gray gradient
```

### Social tab

Explore real-time friend activity:

- **"Hot Right Now"** - Venues with 5+ interested friends (triggers booking agent)
- **Activity timeline** - Friend interests with avatar stacks and timestamps
- **Auto-refresh** - New activities appear every 30 seconds with toast notifications
- **Infinite scroll** - Load historical activity by scrolling down

<Tip>
  To trigger the booking agent: Have 5+ users express interest in the same venue. Switch between demo users and tap the heart icon on a popular venue!
</Tip>

### Profile tab

Manage your account:

- **Demo user switcher** - Test different personas (Alex, Jordan, Morgan, Sam)
- **Interested venues grid** - All venues you've favorited with lazy-loaded images
- **Action items** - Booking coordination tasks (complete or dismiss)
- **Dark/Light mode** - Toggle theme with persistent preferences

## Troubleshooting

<AccordionGroup>
  <Accordion title="Backend won't start - Port already in use">
    Another process is using port 8000. Either stop that process or use a different port:

    ```bash
    # Check what's using port 8000
    lsof -i :8000
    
    # Kill the process (replace PID with actual process ID)
    kill -9 <PID>
    
    # Or use a different port
    uvicorn main:app --reload --port 8001
    ```

    If you change the port, update `baseURL` in `APIService.swift` accordingly.
  </Accordion>

  <Accordion title="iOS app can't connect to backend (physical device)">
    Physical devices can't use `localhost`. You need your Mac's local IP:

    1. Open **System Settings → Wi-Fi → Details** (click the "i" icon)
    2. Copy your IP address (e.g., `192.168.1.45`)
    3. Update `APIService.swift`:
       ```swift
       private let baseURL = "http://192.168.1.45:8000"
       ```
    4. Rebuild the app (Cmd+B) and run

    <Warning>
      Both devices must be on the same Wi-Fi network!
    </Warning>
  </Accordion>

  <Accordion title="Database errors or missing data">
    Reset the database and re-seed:

    ```bash
    # Stop the server (Ctrl+C)
    
    # Delete existing database
    rm luna.db
    
    # Restart server (will auto-seed on startup)
    uvicorn main:app --reload --port 8000
    ```

    You should see log messages confirming database initialization:
    ```
    INFO: Starting Luna Backend API...
    INFO: Seeding database with initial data...
    INFO: Created 4 users
    INFO: Created 20 venues
    INFO: Luna Backend API ready
    ```
  </Accordion>

  <Accordion title="Xcode build errors">
    Common fixes:

    **"No such module 'Foundation'"**  
    Clean build folder: **Product → Clean Build Folder** (Cmd+Shift+K)

    **"iOS 17.0 required"**  
    Update your simulator or device to iOS 17.0+

    **"Signing requires a development team"**  
    Select your Apple ID in **Signing & Capabilities** → Team
  </Accordion>

  <Accordion title="Empty screens or no data loading">
    Check the Xcode console for API errors:

    1. Run the app in Xcode (Cmd+R)
    2. Open the debug console (Cmd+Shift+Y)
    3. Look for errors like:
       - `Network error: Unable to connect to server`
       - `Invalid URL`
       - `Request timed out`

    **Common causes:**
    - Backend server not running
    - Wrong `baseURL` in `APIService.swift`
    - Firewall blocking local connections

    **Verify backend is running:**
    ```bash
    curl http://localhost:8000/venues
    ```

    Should return JSON with venue data.
  </Accordion>
</AccordionGroup>
