## Welcome to Luna

Luna is a production-ready, full-stack iOS application for discovering venues with intelligent recommendations, real-time social feeds, and automated booking coordination.

The platform combines Swift/SwiftUI on iOS with a Python/FastAPI backend to enable social venue exploration through intelligent matching algorithms, real-time friend activity tracking, and automated group coordination.

## Intelligent discovery

Luna's recommendation engine uses a multi-factor scoring algorithm that combines:

- **Popularity score** (30% weight) - Normalized interested user count
- **Category match** (25% weight) - Binary match based on user preferences
- **Friend signal** (25% weight) - Normalized count of friends interested
- **Proximity** (20% weight) - Distance-based scoring using Haversine formula

```swift
// Final score calculation
final_score = (popularity × 0.30) + (category_match × 0.25) + 
              (friend_count × 0.25) + (proximity × 0.20)
```

The **For You** tab displays personalized recommendations with transparent scoring breakdowns, match meter visualizations, and "Why This?" callout cards explaining the reasoning behind each suggestion.

## Real-time social

The **Social** tab provides:

- **"Hot Right Now"** section highlighting venues with 5+ interested friends
- Real-time activity timeline with avatar stacks and timestamp formatting
- Auto-refresh polling (30-second intervals) with new activity toast notifications
- Infinite scroll pagination for historical activities
- Interest activity cards with venue previews and friend avatars

## Booking agent

When 5 or more users express interest in a venue, Luna's booking agent automatically:

1. Detects the interest threshold being reached
2. Creates persistent action items for all interested users
3. Generates unique action codes for coordination
4. Displays toast notifications across the app
5. Adds items to user profiles for tracking and management

<Note>
  The booking agent is configurable and can be adjusted based on venue capacity or day/time in production deployments.
</Note>

## Architecture

Luna follows a modern, production-ready architecture:

### iOS app (SwiftUI + MVVM)

- **Views layer**: 11 screens, 20+ reusable components
- **ViewModels**: AppState singleton + 6 dedicated ViewModels
- **Services layer**: API, Auth, Analytics, Notifications, Image Cache, Persistence
- **Zero external dependencies**: Pure SwiftUI/Combine leveraging native frameworks

### Backend (FastAPI + SQLAlchemy)

- **9 REST endpoints**: Venues, interests, recommendations, social feed, users, action items
- **Async SQLAlchemy ORM** with SQLite persistence
- **Pydantic v2 validation** with custom validators
- **CORS middleware** for iOS integration
- **Database seeding** with 20+ venues and 4 demo users

```swift iOS - API Service
// APIService.swift - Base URL configuration
private let baseURL = "http://localhost:8000"

// Generic request handler with type-safe decoding
func performRequest<T: Decodable>(
    path: String,
    method: String = "GET"
) async throws -> T {
    guard let url = URL(string: "\(baseURL)\(path)") else {
        throw APIError.invalidURL
    }
    
    var request = URLRequest(url: url)
    request.httpMethod = method
    
    let (data, response) = try await URLSession.shared.data(for: request)
    
    guard let httpResponse = response as? HTTPURLResponse else {
        throw APIError.unknown
    }
    
    guard httpResponse.statusCode == 200 else {
        throw APIError.serverError(
            statusCode: httpResponse.statusCode,
            message: "Request failed"
        )
    }
    
    return try JSONDecoder().decode(T.self, from: data)
}
```
